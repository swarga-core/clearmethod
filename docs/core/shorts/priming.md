# Прайминг контекста

Документ описыват этапы прайминга - первоначальной загрузки контекстного окна ИИ-агента стартовым контентом.

Начальная точка прайминга - команда `\cm-prime`. Она либо вызывается вручную при открытии нового чата, либо зашита в правила IDE, и вызывается автоматически.

Эта команда ищет файл `.cm/priming.yml` и начинает читать документы, список которых находится в поле `primers`.

# Формирование прайминга

Конфиг `.cm/priming.yml` формируется на этапе сетапа ClearMethod в проект. Каждое расширение, которое сетапится в проект, добавляет свои праймеры в конфиг. Праймеры могут формироваться интерактивно, при взаимодействии с пользователем.

Праймер - это документ, предназначенный для ИИ-агента, который должен быть прочитан им на этапе прайминга.

Также, при отдельном сетапе расширения в проект, оно автоматически добавляет в конфиг свои праймеры.

# Пример priming.yml

```yaml
primers:
  - core:
    - .cm/core/primer.md
    - .cm/context/rules.md
    - .cm/context/arch.md
  - fsd:
    - .cm/extensions/fsd/primer.md 
```
Праймеры внутри конфига группируются по расширениям, чтобы можно было легко управлять имим (добавлять/удалять)
