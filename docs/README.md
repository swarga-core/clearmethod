# ClearMethod

**Методология структурированной работы с AI в любых процессах**

ClearMethod — это открытая методология и фреймворк для управления AI-агентами в структурированных процессах. Вместо хаотичного взаимодействия с AI через промпты, ClearMethod предлагает систему рабочих процессов, контекстов и проверок качества, которая делает работу с AI предсказуемой и надежной.

Примеры в документации фокусируются на разработке программного обеспечения, но принципы ClearMethod универсальны и применимы к аналитическим, управленческим, научным и другим задачам, требующим структурированного взаимодействия с AI.

---

## Проблема

Разработка с AI-ассистентами сегодня выглядит так:
- Длинные промпты без структуры
- Потеря контекста между сессиями
- Непредсказуемое качество результата
- Отсутствие истории принятых решений
- Сложность передачи работы между разработчиками

В результате AI превращается либо в игрушку, либо в источник технического долга.

---

## Решение

ClearMethod предлагает принципиально новый подход: AI работает не через свободные промпты, а как исполнитель структурированных инструкций на языке CML (ClearMethod Language).

### AI как мета-рантайм

ClearMethod работает как **мета-рантайм**: управляющий слой, который может интерпретировать инструкции напрямую через AI, делегировать исполнение скриптам, использовать workflow engines или интегрироваться с внешними системами. Runtime эволюционирует вместе с проектом от прототипа к production.

Вместо того чтобы "общаться" с AI в чате, вы пишете декларативные инструкции на CML. AI читает, интерпретирует и исполняет их. Это обеспечивает:

- **Прозрачность** — вы видите, что именно будет выполнено
- **Воспроизводимость** — одинаковые инструкции дают предсказуемый результат
- **Передаваемость** — любой AI-агент может продолжить работу по той же инструкции
- **Эволюционность** — runtime адаптируется под задачу без переписывания проекта

### CML: декларативный язык на базе естественного

CML (ClearMethod Language) — это YAML-based DSL, где инструкции пишутся почти на естественном языке, но при этом структурированы и однозначны. AI понимает CML и выполняет команды, не додумывая и не импровизируя сверх меры.

### Пять ключевых концептов

**TASK** — самодостаточная единица работы. Задача содержит всё необходимое для возобновления работы: текущее состояние, историю переходов, принятые решения, артефакты. Любой разработчик или AI-агент может подхватить задачу в любой момент без потери контекста.

**WORKFLOW** — управляемая траектория. Рабочий процесс описан как граф состояний с четкими переходами, предусловиями и постусловиями. AI перемещается по графу, выполняя определенные действия на каждом этапе.

**AGENT** — AI-агент как явная сущность. Информация о модели, управление токенами контекстного окна, осознание возможностей и ограничений.

**CONTEXT** — эффективное управление информацией. AI получает только релевантный контекст для текущего этапа работы. Не больше, не меньше.

**EVENT** — система событий для интеграции. Пакеты реагируют на изменения состояния без жёстких зависимостей друг от друга.

### Интеграция в проект

Фреймворк интегрируется в конкретный проект через интерактивную настройку, создавая папку `.cm/` с конфигурацией и пакетами. Это дает:

- Конфигурация версионируется вместе с кодом
- Команда работает с единой настройкой
- Каждый проект может использовать свой набор пакетов
- Изоляция между проектами
- Легкая адаптация под специфику проекта

На базе этих принципов построена расширяемая архитектура пакетов, которая позволяет добавлять Git-интеграцию, контроль качества, визуализацию процесса и другие инструменты по мере необходимости.

## Ключевые особенности

**Воспроизводимость** — любой разработчик или AI-агент может подхватить задачу на любом этапе благодаря документированному состоянию.

**Качество** — встроенные механизмы проверок (quality gates) не позволяют AI генерировать код без тестов, документации и соответствия стандартам.

**Контроль** — человек всегда принимает финальные решения. AI предлагает варианты, объясняет логику, но не действует автономно в критических ситуациях.

**Гибкость** — минималистичное ядро и модульная архитектура. Используйте только те пакеты, которые нужны вашему проекту.

**Мультисессионность** — работа разбивается на этапы (stages), каждый из которых выполняется в чистом контексте. Это предотвращает деградацию качества при длительных сессиях.

**Декларативность** — рабочие процессы описываются на CML, языке инструкций для AI. Это делает процесс прозрачным и воспроизводимым.

**Прогрессивная реализация** — начинайте с AI-инструкций для быстрого прототипа, постепенно заменяйте критичные части на bash/Python/any-lang для производительности, добавляйте LangGraph для сложной оркестрации или MCP для нативных интеграций. Все подходы сосуществуют в одном проекте.

## Для кого это

**Solo-разработчики** — структура и дисциплина без бюрократии крупных методологий.

**Команды** — единый процесс работы с AI, передача контекста между участниками, code review с AI в рамках общих стандартов. Конфигурация проекта версионируется и разделяется всей командой.

**Enterprise** — контроль качества, audit trail действий AI, соответствие compliance требованиям, возможность работы с конфиденциальными данными.

**Исследователи** — не только разработка ПО. Workflow-подход применим к любым процессам: от написания научных статей до проведения экспериментов.

## Быстрый старт

```bash
# Установка ClearMethod (концептуально, реальная реализация в разработке)
npm install -g clearmethod

# Инициализация проекта
cm init

# Создание первой задачи
cm task create "Добавить аутентификацию пользователей"

# Запуск workflow
cm start
```

## Архитектура

ClearMethod построен на принципе минимального ядра и расширяемой экосистемы пакетов.

**Core** — базовая инфраструктура без domain-specific логики. Только механизмы для работы с задачами, workflow и событиями.

**Concepts** — абстрактные интерфейсы (TASK, WORKFLOW, AGENT, CONTEXT, EVENT — ядро; VCS, QA_GATE, KANBAN — расширения). Определяют "что можно делать", но не "как именно". Описываются на CML.

**Implements** — конкретные реализации концептов (file-task, git-vcs, qa-gates). Именно они выполняют реальную работу.

Такая архитектура позволяет комбинировать пакеты под нужды проекта: от минимального набора для solo-разработчика до полного стека для enterprise-команды.

Подробнее: [Архитектура ClearMethod](03-архитектура/обзор.md)

## Экосистема пакетов

**Реализованные (MVP)**
- `file-task` — хранение задач в файловой системе
- `basic-events` — система событий для интеграции компонентов
- `sbd` — Stage-Based Development (поэтапная разработка)
- `git-vcs` — интеграция с Git
- `qa-gates` — контроль качества (линтеры, тесты, coverage)
- `file-kanban` — визуализация задач в виде Kanban-доски

**Планируемые**
- `smart-context` — умное управление контекстом AI
- `auto-docs` — автоматическая генерация документации
- `code-review` — AI-powered code review
- `test-runner` — интеллектуальное выполнение тестов
- `notifications` — система уведомлений (Slack, email, console)

Полный список: [Обзор пакетов](05-пакеты/обзор.md)
