# Для кого ClearMethod

ClearMethod приносит пользу в разных контекстах: от solo-разработчика до крупной организации. Но методология не универсальна — где-то она критична, где-то полезна, а где-то избыточна. Разберемся честно, для кого и когда ClearMethod имеет смысл.

---

## Solo-разработчики

### Когда полезно

Вы работаете над проектами, которые занимают больше одной сессии. Возвращаетесь к задаче через день, неделю или месяц и тратите время на восстановление контекста: "что я делал, что решил, почему выбрал этот подход". С AI работаете регулярно, но каждый раз объясняете контекст заново.

### Что дает ClearMethod

**Документированное состояние.** Каждая задача сохраняет свою историю. Открыли через неделю — видите текущий этап, выполненные действия, принятые решения. Не нужно искать в истории чата или в коммитах.

**Воспроизводимые процессы.** Описали workflow для типовых задач один раз — используете для всех похожих случаев. Feature, bugfix, refactoring выполняются по единой логике, качество предсказуемо.

**Личная база знаний.** ClearMethod помогает формализовать ваш опыт: какие этапы проходит задача, какие проверки обязательны, какие решения работают. Это становится вашей методологией, адаптированной под ваш стиль.

### Сценарий

Разработчик ведет SaaS-проект в одиночку. Работает вечерами после основной работы. Задача "Добавить интеграцию с Stripe" занимает 4 вечера с перерывами. Без ClearMethod: каждый вечер восстанавливает контекст из коммитов и заметок. С ClearMethod: открывает задачу, видит текущий этап (implementing), понимает что сделано (API клиент, тесты), что дальше (webhook обработка). Продолжает с того места, где остановился.

### Начальные затраты vs выгода

Первая настройка займет 1-2 часа: инициализация проекта, выбор минимальных пакетов, описание базового workflow. Окупается через 3-4 задачи, когда вы перестаете тратить время на восстановление контекста и повторное объяснение процессов AI.

## Команды разработки (2-10 человек)

### Ключевые преимущества

**Единый процесс.** Вся команда работает с AI по одинаковым правилам. Code review, тестирование, документация выполняются по единым стандартам независимо от того, кто делает задачу.

**Передача контекста.** Один разработчик начал задачу, другой продолжил. Без потери контекста, без необходимости пересказывать что было сделано. Состояние задачи документировано, история решений сохранена.

**Версионируемая конфигурация.** Настройки ClearMethod лежат в `.cm/` и версионируются вместе с кодом. Новый участник клонирует репозиторий — получает готовую настройку. Изменения в процессах проходят через PR, обсуждаются командой.

**Onboarding.** Новичок видит не только код, но и процессы: как задачи проходят через этапы, какие проверки обязательны, какие решения принимались раньше. ClearMethod становится частью знаний команды.

### Сценарий

Команда из 5 разработчиков делает финтех-приложение. Высокие требования к качеству и безопасности. Настроили ClearMethod с пакетами: git-vcs (автоматические коммиты по этапам), qa-gates (обязательные тесты и линтер), file-kanban (визуализация задач). 

Разработчик A начал задачу "Реализовать 2FA", прошел этап проектирования, создал design document. Заболел. Разработчик B подхватил задачу: открыл task, прочитал design, увидел текущий этап (implementing), продолжил работу. Все решения A задокументированы, контекст не потерян.

При code review команда использует AI с общими правилами из `.cm/`. Все review проходят по единому чеклисту: security, tests, documentation. Качество стабильно независимо от автора.

## Enterprise / крупные команды

### Критические требования

В крупных организациях работа с AI сталкивается со специфическими вызовами: compliance, аудит, конфиденциальность данных, стандартизация процессов. ClearMethod предоставляет инструменты для решения этих задач.

### Compliance и audit

**Audit trail.** Каждое действие AI документируется: что было выполнено, когда, на каком этапе, с каким результатом. История хранится в структурированном виде, доступна для аудита.

**Quality gates.** Обязательные проверки встроены в workflow. AI не может пропустить тесты, code review, security scan. Если проверка не прошла — переход на следующий этап невозможен.

**Версионирование процессов.** Изменения в workflow проходят через review и версионируются. Видна история: когда процесс изменился, кто принял решение, почему.

### Конфиденциальность данных

**PII protection.** Пакеты для работы с конфиденциальными данными: автоматическое обнаружение PII, санитизация перед отправкой AI, блокировка sensitive файлов.

**Локальная конфигурация.** Секреты и credentials не покидают локальную машину. Mapping хранится локально в зашифрованном виде.

**Контроль отправки данных.** Явный контроль что отправляется AI. Возможность работы с локальными моделями для критичных данных.

### Масштабирование процессов

**Единые стандарты.** Десятки или сотни разработчиков работают по единым процессам. ClearMethod описывает эти процессы явно, делает их воспроизводимыми.

**Переиспользование workflow.** Типовые процессы (feature development, security fix, incident response) описаны один раз, используются всеми командами.

**Кастомизация под домен.** Финтех, healthcare, e-commerce имеют специфические требования. ClearMethod позволяет создавать domain-specific пакеты и workflow.

### Сценарий

Банк с командой разработки 50+ человек внедряет ClearMethod. Требования: все действия AI должны логироваться, customer data не может отправляться внешним API, обязательна security review для любых изменений.

Настройка: базовые пакеты + custom пакет `bank-compliance` с правилами для финтеха, локальная LLM для работы с конфиденциальными данными, qa-gates с обязательным security scan. Workflow включает дополнительный этап: security review перед завершением.

Результат: разработчики работают с AI структурированно, compliance требования выполняются автоматически, аудиторы имеют полную историю действий AI.

## Исследователи и нетехнические специалисты

### Применение вне разработки ПО

ClearMethod спроектирован как domain-agnostic фреймворк. Ядро не знает про код, Git или линтеры — это добавляют пакеты. Базовые концепты (TASK, WORKFLOW, CONTEXT) применимы к любым структурированным процессам: аналитика, управление проектами, научные исследования, написание документов, обработка данных и многое другое.

### Структурированная работа с документами

**Написание статей.** Workflow: сбор материалов → структурирование → черновик → review → финализация. AI помогает на каждом этапе, состояние сохраняется, можно вернуться после перерыва.

**Исследования.** Workflow: формулировка гипотезы → сбор данных → анализ → выводы → отчет. История решений документирована, воспроизводимость обеспечена.

**Работа с данными.** Workflow: очистка данных → exploratory analysis → моделирование → валидация → документирование. Каждый этап с проверками и артефактами.

### Примеры использования

Исследователь пишет научную статью с помощью AI. Описывает workflow: литобзор → методология → эксперименты → результаты → обсуждение → выводы. AI помогает на каждом этапе: ищет релевантные источники, структурирует материал, генерирует черновики разделов. Состояние статьи документировано, можно вернуться к работе через неделю и продолжить с того этапа, где остановились.

Аналитик обрабатывает датасет. Workflow: загрузка → очистка → анализ → визуализация → отчет. На каждом этапе AI выполняет действия, результаты сохраняются. Коллега может подхватить работу, видя что уже сделано.

## Когда ClearMethod избыточен

Методология не универсальна. Есть ситуации, где ClearMethod создает overhead без реальной пользы.

**Разовые эксперименты.** Пробуете новую библиотеку, тестируете идею за 30 минут. Структура здесь замедлит, а не поможет. Обычный чат с AI эффективнее.

**Обучение программированию.** Студент решает учебные задачи. Важна скорость feedback, а не воспроизводимость процесса. ClearMethod добавит сложности без пользы.

**Простые утилиты.** Скрипт на 20 строк для одноразового преобразования данных. Быстрее написать через промпт, чем настраивать workflow.

**Rapid prototyping.** Хакатон, MVP за выходные, proof of concept. Когда скорость критична, а качество и поддерживаемость — потом, ClearMethod может мешать.

**Творческое программирование.** Generative art, эксперименты с алгоритмами, исследовательское программирование. Здесь важна свобода импровизации, структура ограничивает.

Впрочем, даже для быстрых задач базовый ClearMethod (без дополнительных пакетов и сложных workflow) может дать пользу — если вы хотите сохранить историю работы или можете вернуться к задаче позже.

## Самодиагностика

Ответьте на вопросы, чтобы понять нужен ли вам ClearMethod:

**Длительность задач.** Ваши задачи занимают больше одной сессии с AI? Вы возвращаетесь к работе после перерывов?

**Передача контекста.** Нужно ли передавать задачи коллегам? Работаете ли в команде, где несколько человек могут взяться за одну задачу?

**Повторяемость.** Есть ли у вас типовые задачи, которые решаются похожим образом? Хотите ли описать процесс один раз и применять многократно?

**Качество.** Важна ли воспроизводимость результата? Есть ли требования к тестам, документации, code review?

**История решений.** Возвращаетесь ли вы к задачам через недели/месяцы и тратите время на восстановление контекста?

**Compliance.** Есть ли требования к аудиту действий AI, работе с конфиденциальными данными, соблюдению стандартов?

**Если 3+ ответа "да"** — ClearMethod, скорее всего, принесет пользу.  
**Если 1-2 ответа "да"** — попробуйте минимальную конфигурацию и оцените.  
**Если 0 ответов "да"** — возможно, вам пока не нужна дополнительная структура.
